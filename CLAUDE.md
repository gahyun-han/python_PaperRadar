# PaperRadar — Claude Code 규칙

## 코드 변경 시 필수 절차

**어떤 `.py` 파일이든 수정하면, 반드시 아래 순서를 완료해야 한다:**

1. **테스트 파일 작성** — 변경된 로직을 커버하는 pytest 테스트 케이스를 `tests/` 아래에 생성 또는 업데이트한다.
   - 외부 API(arxiv, Zotero, NotebookLM 등)는 mock으로 대체한다.
   - 정상 경로(happy path) + 경계값 + 에러 케이스를 포함한다.
2. **테스트 실행** — `.venv/bin/python3 -m pytest tests/ -v` 로 전체 통과를 확인한다.
3. **실패 시 즉시 수정** — 테스트가 실패하면 코드 또는 테스트를 수정하고 재실행한다. 실패 상태로 작업을 마치지 않는다.

> 사용자에게 "완료했습니다"라고 보고하기 전에 테스트 통과가 전제되어야 한다.
